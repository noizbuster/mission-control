# PRD: TUI input area: visual separation from surrounding context

| Field | Value |
| --- | --- |
| Status | draft |
| Scope | Input area visual treatment (cyan `>` prompt, surrounding spacing) |

## Background

The input area lacked visual separation from the output above and the status bar below, hurting at-a-glance scannability of where typed text would land.

## Goals

- Input area is visually distinct from output above and status bar below.

## Non-Goals

- (none stated)

## Requirements

1. The input area has clear visual separation from the output region above.
2. The input area has clear visual separation from the status bar below.
3. At-a-glance scannability of where typing will land is improved.

## Acceptance Criteria

- User can identify the input area from a static screenshot without watching it being typed into.
